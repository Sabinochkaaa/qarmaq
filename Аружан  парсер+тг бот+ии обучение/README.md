# Qarmaq — мониторинг Telegram-каналов на подозрительный контент

## Состав проекта
- `normalize.py` — убирает эмодзи и разворачивает "стилизованный" юникод (ᴄᴀᴩᴀʙᴀɴ-стиль) в обычные буквы
- `prefilter.py` — keyword-фильтр: решает, стоит ли тратить запрос к Groq на это сообщение
- `groq_classifier.py` — финальная AI-классификация (категория + risk_score + объяснение)
- `collector.py` — сборка готового отчёта по одному сообщению (ссылка, отправитель, контакты)
- `storage.py` — SQLite дедуп + локальный fallback-файл, если сервер недоступен
- `sender.py` — отправка отчёта на сервер Сабины (POST + X-Ingest-Token)
- `main.py` — точка входа: бэкфилл истории → live-мониторинг новых сообщений
- `channels.txt` — список каналов для мониторинга (по одному на строку)
- `.env` — все настройки и секреты

## Установка

```bash
pip install -r requirements.txt
```

## Настройка

1. Открой `.env`, впиши:
   - `TG_API_ID`, `TG_API_HASH` — с my.telegram.org
   - `TG_PHONE` — номер аккаунта, под которым уже состоишь в нужных каналах
   - `GROQ_API_KEY` — твой ключ
2. Открой `channels.txt`, замени примеры на реальные `@username` каналов, где ты уже состоишь.

## Запуск

```bash
python3 main.py
```

При первом запуске Telegram попросит код подтверждения в консоли (придёт в сам Telegram) —
это нормально, один раз. После этого сессия сохранится в файл `qarmaq_session.session`,
и повторно код вводить не нужно.

Скрипт сам:
1. Выгружает последние `BACKFILL_LIMIT` сообщений (по умолчанию 300) из каждого канала и анализирует их.
2. После этого переходит в режим live-мониторинга и анализирует каждое новое сообщение сразу при поступлении.
3. По умолчанию (`SEND_ONLY_SUSPICIOUS=true`) на сервер уходят только сообщения с категорией
   отличной от "чистое" — чтобы не засорять базу. Если нужно отправлять вообще всё — поставь `false`.

## Формат отчёта (что уходит на сервер / в reports_fallback.jsonl)

```json
{
  "channel": "Название канала",
  "channel_id": -1001234567890,
  "channel_username": "example_channel",
  "message_id": 4821,
  "message_link": "https://t.me/example_channel/4821",
  "message_date": "2026-06-22T10:15:00+00:00",
  "sender_id": 123456789,
  "sender_type": "user",
  "extracted_contacts": [],
  "raw_text": "исходный текст сообщения",
  "normalized_text": "текст после очистки эмодзи и обфускации",
  "category": "утечка_бд",
  "risk_score": 0.91,
  "explanation": "Сообщение прямо рекламирует продажу базы данных по номеру телефона.",
  "prefilter_candidate": true,
  "prefilter_hints": ["утечка_бд"],
  "analyzed_at": "2026-06-22T10:16:03+00:00"
}
```

Поле `sender_type`:
- `"user"` — отправил конкретный человек, `sender_id` — его Telegram ID
- `"channel_post"` — пост от имени самого канала/бота; в этом случае
  `extracted_contacts` пытается вытащить @юзернеймы/номера из текста самого объявления

## Важные заметки по безопасности

- `.env` содержит реальные секреты (токен, ключи) — не публикуй этот файл и не клади в открытый git-репозиторий.
- При остановке/перезапуске скрипта уже обработанные сообщения не анализируются повторно (SQLite `processed.db`).
- Если сервер Сабины недоступен — отчёты не теряются, падают в `reports_fallback.jsonl`, их можно дозалить позже.
